Introduction back to ToC

Sensors are embedded in multiple Internet of Things (IoT) devices, for example, smart fire detectors, record data that can be used by service technicians for troubleshooting. The observations of these sensors can, however, be anomalous causing unwanted behaviour in different households. Many different components can cause this unwanted behaviour, and these different components affect each other. Based on the functional analysis and available domain knowledge, a combination of an anomaly and failure mode and effect analysis (FMEA) ontology is built for the specific case of IoT devices with sensors. The folio ontology but reuses concepts of both FMEA an anomaly knowledge, describing their interlinkage.
